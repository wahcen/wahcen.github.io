Spring框架-helloworld项目配置

一、使用.xml文件管理对象关系

使用上一个例子中的内容：在项目resources目录中添加applicationContext.xml文件并插入。但是，这种管理方式相较于注解方法非常麻烦，因此，实际开发中大多数使用的还是注解的方式，让spring容器自动管理对象创建以及对象之间的依赖关系。

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--
        bean元素，描述当前对象需要由spring容器管理
        id属性，标识对象
        class属性，被管理类名
    -->
    <bean id="service" class="hello.MessageService"></bean>

    <!-- property属性中name标签对应MessagePrinter类中的私有成员变量
        service，ref指向上面id=service的对象，表示注入该对象 -->

    <bean id="printer" class="hello.MessagePrinter">
        <property name="service" ref="service"></property>
    </bean>
</beans>

二、引入log4j支持

引入log4j可以简化控制台输出，在resources文件夹新建log4j.properties文件,log4j依赖会默认扫描项目根目录下的log4j.properties文件以配置自身。示例配置如下，配置方法见[log4j文档]([http://logging.apache.org/log4j/1.2/faq.html#noconfig](http://logging.apache.org/log4j/1.2/faq.html#noconfig)

### direct log messages to stdout ###
log4j.appender.stdout=org.apache.log4j.ConsoleAppender
log4j.appender.stdout.Target=System.err
log4j.appender.stdout.layout=org.apache.log4j.PatternLayout
log4j.appender.stdout.layout.ConversionPattern=%d{ABSOLUTE} %5p %c{1}:%L - %m%n

### direct messages to file mylog.log ###
log4j.appender.file=org.apache.log4j.FileAppender
log4j.appender.file.File=c\:mylog.log
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d{ABSOLUTE} %5p %c{1}:%L - %m%n

### set log levels - for more verbose logging change 'info' to 'debug' ###

log4j.rootLogger=debug, stdout

三、引入junit单元测试支持

若使用maven管理项目则直接在pom.xml中添加junit依赖即可，否则需要自己导入所需要的jar包

<!--        添加junit 单元测试依赖-->
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
        </dependency>

<!--        添加spring-test依赖 -->
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-test</artifactId>
            <version>5.1.8.RELEASE</version>
        </dependency>

导入后即可在test文件夹下新建测试类，仅需在需要测试的方法上添加@Test注解即可run该方法
